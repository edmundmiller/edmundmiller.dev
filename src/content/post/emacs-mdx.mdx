My [Astro setup for Doom Emacs](/posts/emacs-astro/) handled `.astro` files, but it did not assign a useful major mode to `.mdx` files. I wanted Markdown editing features while working on posts that mix prose with components.

I adapted the approach from Tailscale's guide to [configuring Emacs for MDX](https://tailscale.dev/blog/configuring-emacs-mdx). This is the configuration in my Doom Emacs Astro module:

```emacs-lisp title="modules/lang/astro/config.el"
;; MDX support
(add-to-list 'auto-mode-alist '("\\.\\(mdx\\)$" . markdown-mode))

(when (modulep! +lsp)
  (add-hook 'markdown-mode-local-vars-hook #'lsp! 'append))
```

The mode association makes Emacs open `.mdx` files with `markdown-mode`. In my setup, that restored Markdown syntax highlighting and editing commands. The guarded hook also asks Doom to start LSP when the Astro module has `+lsp` enabled.

This is a small compatibility fix, not complete MDX language support. `markdown-mode` does not understand every JSX expression or component boundary. The hook also applies to every Markdown buffer, and it does not install an MDX language server. Completion and diagnostics still depend on an LSP server that recognizes the file.

For editing prose and fenced code in this site, the simpler mode association solved the immediate problem. More precise JSX-aware editing would need a dedicated MDX mode or language-server configuration.
